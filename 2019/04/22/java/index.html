1.overriding & overloading

(achieve Polymorphism)

overriding

In oop, overriding allows subclass to provide a specific implementation of a method that is already provided by one of its super classes.

overloading

Allow different methods to have the same name, but different signatures differing on the number or type of the input parameters. Overloading cannot distinguish two same signatures with different return type.
Therefore, the return type can be same or different. Just make sure the number or type of input parameters are different.

static & final

static

variable, method, class
static variable and method are on the class level which is shared with the objects.

final

  • variable: cannot change the value of final variable
  • method: cannot override final method, final method is inherited
  • class: cannot extend final class

1)blank final varialbe:
A final variable that is not initialized at the time of declaration. It can be initialized only in constructor

2)static blank final variable:
static final variable that is not initialized at the time of declaration. It can be initialized only in static block.

Access modifier

restrict the scope of class, method, variable.
4 types: default(no keyword), private, protected, public.

Interface

(achieve polymorphism)

  • only contain method signatures and fields, cannot contain the implementation of the methods.
  • public or default access modifier.
  • class that implements an interface must implement all the methods declared in the interface. Doesn’t need to
    declare the variables of the interface. Only the methods.
  • a class can implement multiple interfaces
  • interface cannot be instantiated
  • interface can inhert another interface

Abstract

abtract class is similar to interface except that it can contain method implementation.
It provides common mehtod implementation to all the subclasses or to provide default implementation.

  • abstract class cannot be instantiated
  • abstract method doesn’t have a body
  • if a class has abstract methods, then the class is abstract class using ‘abstract’ keyword
  • abstract class can have no abstract methods
  • subclass of abstract class must implement all the abstract methods unless the subclass is also an abstract class.
  • abstract class can implement interfaces without providing implementation of interface methods.
  • abstract class can have constructor

Four Pillars

  • Encapsulation
    data hiding, wrap variable and methods in a single unit.
  • Polymorphism
    method overriding and overloading
  • Abstraction
    detail hiding, only show the relevant details, reduce complexity.
    achieved by interface and abstract classes.
  • Inheritance

class cannot inherit multiple classes

JVM

ava virtual machine is a run-time engine to run Java applications, it is a interpreter by executing java code.

1) diff between JDK, JRE, JVM

2) heap space and stack memory
Java heap stores objects and classes;
Java stack memory is used for execution of a thread. It contains references to the objects in the heap.

  • Objects stored in heap are globally accessible; stack memory can’t be accessed by other threads.
  • heap memory lives during the application execution; Stack memory is short-lived.
  • Stack memory size is less than to Heap memory, but faster

3) Garbage Collection
Runs on the heap memory to free the memory used by objects that doesn’t have reference.

JVM heap memory is divided into two parts - Young generation and old generation.

When the young generation is filled, garbage collection is performed called Minor GC. Old Generation memory contains the objects that are long-lived and survived after many rounds of Minor GC. When old generation memory is full, Major GC performs.
